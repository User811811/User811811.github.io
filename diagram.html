Pixel Integration Flow

The standard integration relies on tvScientific uploading a CSV of all tracked events to a dedicated FTP endpoint for processing by Impact.

1. FTP Setup

Client creates a dedicated FTP event delivery in Impact for tvScientific.

2. Event Tracking

tvScientific tracks all potential conversion events for the client.

3. Data Upload

tvScientific uploads a complete CSV of events to the FTP endpoint.

4. Processing

Impact ingests the CSV, attributes events, and calculates payout.

Postback Integration Flow

This enhanced method adds a real-time postback from Impact to tvScientific. This helps tvScientific reconcile data and upload a cleaner, more accurate CSV file, reducing duplication.

1. Impact Tracks Event

A conversion is tracked on Impact's side via a standard tracking pixel.

2. Real-time Postback

Impact sends an instant server-to-server postback to tvScientific confirming the event.

3. Reconciled Upload

tvScientific uploads a reconciled CSV containing only relevant, verified events.

4. Final Processing

Impact processes the cleaner CSV for accurate payout.

UTT Integration Flow

The most robust method, functionally identical to the Postback flow, but initiated via Impact's Universal Tracking Tag (UTT). This is for clients using the Partner Integrations Hub to manage partner code.

1. UTT Tracks Event

A conversion is tracked via the client's UTT, which hosts tvScientific's code through the Partner Integrations Hub.

2. Real-time Postback

Impact sends an instant server-to-server postback to tvScientific confirming the event.

3. Reconciled Upload

tvScientific uploads a reconciled CSV containing only relevant, verified events.

4. Final Processing

Impact processes the cleaner CSV for accurate payout.
